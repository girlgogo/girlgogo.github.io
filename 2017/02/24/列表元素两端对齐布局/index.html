列表元素的两端对齐布局

发表于 2017-02-24 | 分类于 CSS 技巧

display:inline-block/text-align:justify下列表的两端对齐布局

1、何为列表元素的两端对齐布局

定义：
所谓列表元素的两端对齐就是每行列表元素的第一个元素与父容器的左边缘重合，最后一个元素与父容器的右边缘重合。

2、如何实现元素的两端对齐

CSS2中text-align有一个属性值为justify，为对齐之意。其实现的效果就是可以让一行文字两端对齐显示（文字内容要超过一行）。

text-align 主要用来控制inline水平元素、inline-block元素的对齐与显示的，例如嵌套行内标签的文字、图片、input表单控件等；而对block水平的元素是没有作用的。

3、目前列表元素如何实现两端对齐的

1、首先是淘宝网首页热卖单品的例子

其列表布局使用的是传统的浮动(float)布局，通过width属性强行增大父容器的宽度来实现看上去的“两端对齐”效果的。

2、人人网热门分享的两端对齐效果的实现方法，它的列表布局为inline-block布局。

其也是通过增大父标签的宽度来实现看上去的“两端对齐”效果的，不过其不是通过width属性来增加父标签的宽度的，而是使用的margin负值。

3、利用white-space: nowrap，此方法需在inline-block布局基础上使用，一般用在实现单行列表元素看上去的“两端对齐”效果上。

4 text-align:justify实现两端对齐的好处

好处：简单方便。
只要一个text-align:justify声明，里面的元素就自动等间距两端对齐布局！无需计算每个列表元素间的margin间距，更不用去修改父容器的宽度。

6 display:inline-block + text-align:justify下列表的两端对齐布局

1、IE8+、现代浏览器
栗子：li列表的两端对齐

注意：

  • 列表元素首尾标签留空（或换行）

  • 对于 IE8，列表元素不能处于font-size:0的环境下，至少code>font-size:1px，因为IE8浏览器font-size:0或直接把换行空格或普通空格抹掉而无法实现两端对齐效果。

2、IE6/7 浏览器
如何让IE6/7浏览器也有列表元素支持text-align:justify属性？

  • inline 标签化：就是列表元素需要使用 inline 水平的标签，例如span,a, strong, em等，而li, div这些标签就不可以。
  • 结束标签连续化：指列表元素及其内部标签的结束标签需要连在一起。

注意：

  • 如果列表标签内嵌多层，则所有层级的结束标签都要连续。

E6/IE7 浏览器同时满足inline标签化、结束标签连续化后，再加上先前现代浏览器下的首尾标签留空，便能够实现列表元素的两端对齐！

最后一行的两端对齐
