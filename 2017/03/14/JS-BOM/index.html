08 章 BOM

JavaScript：

  • 核心（ECMAScript），由ECMA-262定义，提供核心语言功能
  • 文档对象模型（DOM），提供访问和操作网页内容的方法和接口
  • 浏览器对象模型（BOM），提供与浏览器交互的方法和接口

8.1 window对象

BOM 的核心对象是 window，表示浏览器的一个实例。

在浏览器中，window 对象的双重角色：
1) 通过 JavaScript 访问浏览器窗口的一个接口
2) ECMAScript 规定的 Global 对象

8.1.1 全局作用域

全局变量
方式
1) 定义全局变量：不能通过 delete 操作符删除，因为[[Configurable]]特性的值为 false
2) 在window对象上定义属性：可以通过delete删除

注意
通过查询 window 对象，知道某个未声明的变量是否存在。

兼容性
Windows Mobile 平台 IE 不允许通过 window.property = value的形式在 window 对象上创建新的属性和方法。

8.1.2 窗口关系及框架

如果页面中包含框架，则每个框架都拥有自己的 window 对象，并且保存在 frames集合中。可以通过数值索引（从0开始，从左到右，从上到下）或者框架名称来访问相应的 window 对象。

框架相关 window 对象属性
1) top：指向最外层的框架（浏览器窗口）
2) parent：指向当前框架的直接上层框架
3) name：除非最高层窗口通过 window.open() 打开，否则window.name不会包含任何值
4) self：始终指向 window ，与 window 对象可以互换使用。引入的目的是为了与 top 和 parent 对象对应起来，不格外包含其他值。

8.1.3 窗口位置

和框架位置相关的属性（无法跨浏览器获得精确坐标）

框架移动方法 (只能对最外层 window 对象使用；Opera和IE7+中默认是禁用的)
1) moveTo()方法：通过坐标移动到新位置，参数（新位置x，新位置y）
2) moveBy()方法：通过设置移动的像素移动到新位置，（水平移动像素数，垂直移动像素数）

8.1.4 窗口大小

页面视图容器：单个标签页对应的浏览器窗口

移动设备的情况比较复杂！

属性 含义 浏览器 实际行为
window.innerWidth／innerHeight 可见视口 可见视口
document.documentElement.clientWidth／clientHeight 可见视口 移动 IE 浏览器 可见视口
document.documentElement 其他移动浏览器 布局视口 渲染后页面的实际大小
document.body.clientWidth／clientHeight 布局视口 移动 IE 浏览器 渲染后页面的实际大小

调整浏览器窗口大小(不适用于框架，只能对最外层window对象使用；Opera和IE7+中默认是禁用的)
1) resizeTo()：参数（新高度，新宽度）
2) resizeBy()：参数（新窗口与原窗口宽度之差，新窗口与原窗口高度之差）

8.1.5 导航和打开窗口

window.open():导航到一个特定的URL或打开一个新的浏览器窗口
参数（4）：要加载的URL、窗口目标、特性字符串（在第二个参数并不是一个已经存在的窗口或框架时才会起作用）、新页面是否取代浏览器历史记录中当前加载页面
返回值：返回一个指向新窗口的引用，通过这个返回的对象可以向操作其它窗口一样操作心打开的窗口
注意：
1) 第三个参数没有传入的话将会根据浏览器的默认设置打开一个默认的新窗口或新标签
2) 有些浏览器在默认情况下可能不允许针对浏览器主窗口调整大小或移动位置，但对window.open()创建的窗口可以
3) 主窗口无法通过close()关闭，但window.open()创建的窗口可以
4) 弹出的窗口通过close()关闭后窗口引用还在，但只能用于监测closed属性

  1. 弹出窗口
    特性字符串（第3个参数）

相关属性和方法

  • closed属性：窗口是否已经关闭布尔值
  • opener属性：保存着指向打开它的原始窗口对象，该值设置为null会使新建的窗口标签在页在独立的进程运行（导致标签页间无法通信）

  1. 安全限制
    注意：打开计算机硬盘上的网页时IE会解除对弹出窗口的某些限制

  2. 弹出窗口屏蔽程序
    屏蔽方式：
    1) 浏览器内置弹出窗口屏蔽程序
    2) 浏览器扩展或其他程序阻止的弹出窗口

检测打开的弹出窗口窗口是否被屏蔽
注意：不会阻止浏览器显示与被屏蔽的弹出窗口有关的消息

8.1.6 间歇调用和超时调用

JavaScript 是单线程语言，但允许通过设置超时值和间歇时间值来调度代码在特定的时刻执行。

window.setTimeout()方法
说明：超时调用，在指定的时间过后执行代码
参数（2）：要执行的代码、以毫秒表示的时间
返回值：一个数值ID，是计划执行代码的唯一标识
注意：

  • 第一个参数传递字符串肯恩工会导致性能损失，因此不建议第一个参数使用字符串
  • 第二个参数指的是JS再过多长时间把当前任务添加带JS任务队列，如果队列非空，和等前面的任务执行完

window.clearTimeout()方法：取消尚未执行的超时调用计划
参数：超时调用ID
注意：超时调用的代码都在全局作用域中执行，因此函数中的 this 值在非严格模式下为 window 对象，严格模式下为 undefined

windwo.setInterval()方法
说明：间歇调用，每隔一段指定的时间就执行一次代码
参数（2）：要执行的代码、以毫秒表示的时间
返回值：一个数值ID，是间歇执行代码的唯一标识
注意：间歇调用每隔一定时间将代码插入任务队列，有可能后一个间歇调用在上一次调用没完成就启动。

window.clearTimeout()方法：终止间歇调用

8.1.7 系统对话框

类型 包含
同步 alert prompt confirm
异步 print find

同步

  • alert():警告对话框
    用途：显示一些用户无法控制的信息，例如错误消息
  • confirm():确认对话框
    用途：让用户决定是否执行给定的操作
  • prompt():提示输入框
    用途：提示用户输入一些文本

异步

  • print():显示“打印”对话框（异步）
  • find()：显示“查找”对话框（异步）

8.2.2 位置操作

location的方法 说明
assign方法 通过传递一个url作为参数，立即打开新URL并在浏览器的历史纪录中生成一条记录
replace方法 用法同assign，但不会留下历史纪录，也就无法通过前进/后退导航。
reload方法 重新加载当前显示的页面
  • assign()：传递一个 URL ，会立即打开新 URL ，并在浏览器的历史记录中生成一条记录。
  • 修改 location 对象的其他属性也可以改变当前加载的页面
  • replace()：用户不能回到前一个页面，不会生成历史记录
  • reload()：重新加载当前显示的页面，最好放在代码的最后一行

兼容性：所有浏览器都支持。

8.3.1 检测插件

1、navigator.plugins 数组：（非 IE 浏览器）
说明：数组中的每一项都包含下列属性。
兼容性：非IE

属性名 说明
name 插件的名字
description 插件的描述
filename 插件的文件名
length 插件所处理的MIME类型数量

2、ActiveXObject 类型 ( IE 专有)
说明：IE不支持Netscape式的插件。只能通过尝试使用专有的ActiveXObject类型创建一个特定插件的实例。
唯一标识符：IE是以COM对象的方式实现插件的，而 com对象使用唯一表示来标识。例如Flash的标识符ShockwaveFlash.ShockwaveFlash。

3、兼容处理

4、navigator.plugins.refresh方法：刷新插件集合
参数：是否重新加载包含页面的所有页面的一个布尔值。

8.3.2 注册处理程序

说明：随着RSS阅读器和在线电子邮件程序的兴起，注册处理程序就为像使用桌面程序一样默认使用这些在线应用程序提供了一种方式。
registerContentHandler() 方法
用途：为一个站点指明它可以处理的特定类型的信息。
参数（3）：要处理的MIME类型、可以处理该MIME类型的页面的URL、应用程序的名称。
限制：firefox4 之前的版本只允许在该方法中使用三种MIME类型

可用的MIME 说明
application/rss+xml RSS
application/atom+xml ATOM
application/vnd.mozilla.maybe.feed feed

举个例子：将一个站点注册为处理RSS源的处理程序

registerProtocolHandler()方法
用途：为一个站点指明它可以处理那种协议的请求。
参数（3）：要处理的协议（mailto、ftp等）、处理该协议个页面的URL、应用程序的名称。
局限：Firefox2实现了该方法，但不能用，Firefox3才将其完整实现。
举个例子：注册一个mailto协议的处理程序，该程序指向一个基于Web的电子邮件客户端。

8.5 history 对象

说明：window对象的属性，每个浏览器窗口、每个标签页乃至每个框架都有自己的 history 对象。
局限：处于安全方面的考虑，开发人员无法得知用户浏览过的 URL。

8.5.1 go方法

用途：在用户的历史纪录中任意跳转。
参数：一个整数值(负数表示向后，整数表示向前)或字符串(跳转到历史纪录中包含改字符串的第一个位置，可能向前也可能向后)

说明：保存着历史纪录的数值。
技巧：对于加载到窗口、标签页或框架中的第一个页面而言，值为0.
