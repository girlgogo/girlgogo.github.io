16.1 跨文档消息传递

简称：XDM
用途：在不同域之间传递消息
现状：XDM已经作为一个规范独立出来，现在叫做Web Messaging
兼容性：IE8+ Firefox3.5+ Safari4+ Opera Chrome ios版Safari Android版webkit

官方页面：http://dev.w3.org/html5/postmsg

核心方法

  • postMessage()方法：向另一个“地方”传递数据
    参数（2）：消息字符串（可以使其它数据结构，但支持不好），允许接收方所在的域字符串（”*”表示可以发给来自任何域的文档）
  • message事件
    触发DOM：window
    触发时机：目标页面接收到消息时
    事件对象重要属性
event对象属性 值 备注
data postMessage()的第一个参数
origin 发送消息文档所在的域 比如”http://www.wrox.com“
source 发送消息的文档的window对象的代理(只能通过这个代理调用postMessage()方法来发送回执信息) 并非实际的window对象

16.2.3 dataTransfer对象

历史：IE5 将其作为事件对象的属性引入
用途：用于在拖放操作时实现数据交换
特点：只能在拖放事件的事件处理程序中使用

主要方法

  • setData()方法：保存键值对
    参数（2）：数据类型，值
    注意：关于数据类型，IE只提供了”text”和”URL”两种；HTML5允许指定各种MIME类型，而且会将’text’映射为’text/plain’，将’URL’映射为’text/uri-list’
  • getData()方法：取得由setData()保存的值
    参数：数据类型
    注意：只能在drop事件中读取

附：解决Firefox第5个版本之前不能正确映射’url’和’text’执行HTML5的映射的问题

16.2.6 其它成员（H5规范）

HTML 5 规范规定 dataTransfer对象还应该包含的下列方法和属性：
方法

  • addElement(element):为拖动操作添加作为拖动源而响应回掉的对象
    参数：拖动源
    兼容性：Firefox3.5
  • clearData(format)：清除以特定格式保存的数据
    参数：特定格式
    兼容性：IE Firefox3.5+ Chrome Safari4+
  • setDragImage(element, x, y)：指定一幅图像，当拖动发生时，显示在光标下面
    参数（3）：
  1. 要显示的HTML元素（不仅可以是img元素）
  2. 光标在图像中显示的x坐标
  3. 光标在图像中显示的y坐标
    兼容性：Firefox3.5+ Safari4+ Chrome

属性

  • types：当前保存的数据类型，一个类似数组的集合，以”text”这样的字符串形式保存着数据类型

兼容性：IE10+ Firefox3.5+ Chrome

16.3 媒体元素

作用：方便地嵌入媒体文件，提供常用功能API，可以自定义控件
相关标签：<video>、<audio>
重要属性

  • src：指向要加载的媒体文件（可以替换为source属性）
  • width、height：视频播放器大小
  • poster: ：指定图像的URL（在加载视频期间显示一幅图像）
  • controls：指浏览器应该显示UI控件，以便用户操作媒体内容

兼容性：IE9+ Firefox3.5+ Safari4+ Opera10.5+ Chrome ios版safari Android版webkit

16.4 历史状态管理

事件

事件名 触发 事件对象属性 用途
haschange URL参数发生变化时触发 在不加载新页面的情况下改变浏览器的URL
popstate 按下“后退”按钮时触发 state:传递给pushState()的事件对象

方法

  • history.pushState()：将新的状态加入历史状态栈
    参数（3）：状态对象、新状态的标题、相对URL（可选）
    特点：新的状态加入历史状态栈后，浏览器地址会变为新的相对URL，location.href也会相应改变，但不会向服务器发送请求
  • history.replaceState()：重写当前状态（不会在历史状态栈中创建新的状态）
    参数：同pushState()

兼容性：Firefox4+ Safari5+ Opera11.5+ Chrome
