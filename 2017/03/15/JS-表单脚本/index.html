14.1 表单的基础知识

  • HTMLFormElement: <form>标签对应的节点

独有属性

属性名 说明 对应的form标签属性
acceptCharset 服务器能够处理的字符集 accept-charset
action 接收请求的URL action
elements 表单中多有控件的集合（HTMLCollection） 无
enctype 请求的编码类型 enctype
length 表单中控件的数量 无
method 要发送的HTTP请求类型（”get”、”method”） method
name 表单的名称 name
target 用于发送请求和接受响应的窗口名称 target

独有方法

  • reset(): 将所有表单域重置为默认值
  • submit(): 提交表单

获取方式

  • 方式一：和其它元素一样
  • 方式二：document.forms: 页面中所有form的集合
  1. 数值索引
  2. name属性值

其它（不推荐）：document.[form的name值]

14.1.1 提交表单

形式一：HTML形式
触发提交:

  • 点击：点击input[type=”submit”]或input[type=”image”]或button[type=”submit”]
  • 回车：表单中有提交按钮而且拥有焦点时按回车

特点：请求发给服务器前触发submit事件
取消提交：submit事件触发时取消事件的默认行为

形式二：submit()方法
触发提交：执行表单节点的submit()方法时
特点：不会触发submit事件（调用方法前先验证表单数据）

注意：防止重复提交表单

  • 针对形式一：第一次提交表单后禁用提交按钮
  • 针对形式二：onsubmit事件处理程序取消后续的表单提交操作

14.1.2 重置表单：所有表单字段都会恢复到页面刚加载完时的初始值

形式一：HTML形式
触发重置: 点击input[type=”reset”]或button[type=”reset”]

特点：重置前会触发reset事件
取消提交：reset事件触发时取消事件的默认行为

形式二：reset()方法
触发提交：执行表单节点的submit()方法时
特点：项单击一样触发reset事件

注意：提供一个取消按钮让用户回到前一个页面，而不是不分青红皂白重置表单中所有的值。

14.1.3 表单字段

  • form.elements属性：包含表单中的所有字段的有序列表
    索引方式一（推荐）：form.elements[下标]
    返回数据：获得对应输入元素的字段
    索引方式二（推荐）：form.elements[“name属性值”]
    返回数据：对应元素的字段或由多个字段组成的NodeList集合
    索引方式三（向后兼容）：form[下标]等同于form.elements[下标]
    索引方式四（向后兼容）：form[”name属性值“]
    返回数据：只返回第一个命名字段
  1. 共有的表单字段属性（fieldset之外）
属性 类型 含义
disabled 布尔值 表示当前字段是否被金禁用
form 引用（只读） 指向当前字段所属表单的指针
name 字符串 当前字段的名称
readOnly 布尔值 表示当前字段是否只读
tabIndex 数字 当前字段的切换序号
type 字符串 当前字段的类型,如”checkbox”、”radio”
value 字符串或文件路径（只读） 当前字段将被提交给服务器的值

注意：

  1. 除了form属性之外的所有其他属性都可以通过JS修改
  2. 避免多次提交表单
  • 不能通过onclick事件处理程序阻止提交，因为有些浏览器会在触发表单的submit事件之前触发click事件，这样表单将永远无法提交；
  • 不适合表单中不包含提交按钮的情况
