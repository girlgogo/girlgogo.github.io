使用 Canvas 绘图

15 章 使用 Canvas 绘图

历史：
由苹果公司推出，主要用在 Dashboard 微件中

兼容性(部分支持)：IE9+ Firefox1.5+ Safari2+ Opera9+ Chrome ios版Safari Android版webkit

15.1 基本用法

注意：
必须先设置<canvas>元素的 width 和 height 属性，指定绘图区域大小
标签中的信息只有当浏览器不支持 canvas 时才会显示

  • canvasEle.getContext()：获得画布的绘图上下文对象
  • canvasEle.toDataURL(MIME)：导出canvas元素上绘制的图像
    参数：图像的MIME类型格式
    注意：
    如果绘制到画布上的图像源来自不同的域会抛出错误
    较新的版本中加入对”image/jpeg”参数的支持（IE9、Firefox3.5和opera10）

15.2 2D上下文

用途：可以绘制简单的2D图形，比如矩形、弧线和路径。

特点：坐标开始于canvas元素左上角，原点坐标是（0，0）

15.2.1 填充和描边

  • context.fillStyle：填充，用指定的样式（颜色、渐变或图像）填充图形
  • context.strokeStyle：描边，只在图形的边缘画线

注意：值可以是CSS指定颜色值的任何格式、渐变对象、模式对象。默认值都是“#000000”。

15.2.2 绘制矩形

矩形是唯一一种可以直接在 2D 上下文中绘制的形状。与矩形有关的三个方法包括：fillRect()、strokeRect()、clearRect()。都接受4个参数：矩形的 x 坐标、y 坐标、宽度、高度。

  • fillRect()：填充矩形区域
  • strokeRect()：为矩形区域描边
    context.lineWidth：描边线条的宽度（任意整数）
    context.lineCap：线条末端平头(“butt”)、圆头(“round”)或方头(“square”)
    context.lineJoin：控制线条相交的方式是圆交(“round”)、斜交(“bevel”)还是斜接(“miter”)
  • clearRect()：清除矩形区域
    共同参数（4）：x坐标、y坐标、宽度、高度

15.2.3 绘制路径

必须

  • context.beginPath()：开始绘制新路径
  • context.stroke()：完成描边
  • context.fill()：完成填充
  • context.clip()：完成剪切

绘制路径

  • arc(x, y, radius, startAngle, endAngle, counterclockwise)：根据圆心等画弧线
    x,y：圆心
    radius：弧线半径
    startAngle：开始角度
    endAngle：结束角度
    counterclockwise：是否按逆时针方向计算起始角度
  • arcTo(x1, y1, x2, y2, radius)：根据起始点画弧线
    x1,y1：起点
    x2,y2：终点
    radius：半径
  • bezierCurvaTo(c1x, c1y, c2x, c2y，x, y)：从上一点开始绘制一条曲线
    c1x, c1y：控制点
    c2x, c2y：控制点
    x,y：终点
  • lineTo(x,y)：从上一点开始绘制一条直线
    x,y：终点
  • moveTo(x,y)：移动绘图游标到指定点（不画线）
    x,y：目的地
  • quadraticCurveTo(cx, cy, x, y)：从上一个点开始绘制一条二次曲线
    cx, cy：控制点
    x, y：终点
  • rect(x, y, width, height)：绘制矩形（矩形路径而不是独立的形状）
    x, y：起点
    width, height：宽度和高度

15.2.4 绘制文本

相关属性(都有默认值)

属性 说明 值 备注
context.font 表示文本样式 CSS中指定字体的格式 例如”10px Arial”
context.textAlign 文本的对齐方式 “statr”/“end”/“left”/“right”/“center” 推荐用”start”/“end”替换”left”/“right”
context.textBaseline 文本的基线 “top”/“hanging”/“middle”/“alphpbetic”/“ideographic”/“bottom”

绘制方法

  • fillText(text, x, y, [maxWidth])：绘制文本填充（使用更多）
  • strokeText(text, x, y, [maxWidth])：绘制文本描边
    共同参数：要绘制的文本字符串，x坐标，y坐标，可选的最大像素宽度
  • measure(text)：根据font、textAlign、textBaseline的当前值辅助确定文本大小的方法
    参数：要绘制的文本
    返回值：TextMetrics对象（该对象目前只有一个width属性）
    用途：在指定大小的区域绘制大小合适的字体

15.2.5 变换

  • 修改变化矩阵的方法：会将坐标系的原点从(0,0)位置移动到指定位置

  • 跟踪上下文状态变化的方法
    save()：将当前的所有状态压入一个栈结构，用于将来通过restore()恢复之
    注意：只保存绘图上下文的设置和变换，不保存绘图上下文绘制的内容
    restore()：在保存设置的栈结构中向前返回一级，恢复之前的状态，连续调用可以一级一级返回

15.2.6 绘制图像

  • drawImage(img, x, y, [width], [height],[partX], [partY], [partWidth], [partHight])：将图像或另一块画布绘制到当前画布上

参数（9）：

  • img：img节点，也可以是另外一个canvas，不能来自其它域
  • x：绘制起点x坐标
  • y：绘制起点y坐标
  • [width]：图片宽度
  • [height]：图片高度
  • [partX]：图片的目标部分的起点x坐标
  • [partY]：图片的目标部分的起点x坐标
  • [partWidth]：图片的目标部分的宽度
  • [partHight])：图片的目标部分的高度

注意：不会影响上下文变换矩阵

15.2.7 阴影

2D 上下文根据下列属性自动为形状或路径绘制出阴影：

  • shadowColor：用 CSS 颜色格式表示的阴影颜色，默认黑色；
  • shadowOffsetX：形状或路径 x 轴方向的阴影偏移量，默认为0；
  • shadowOffsetY：形状或路径 y 轴方向的阴影偏移量，默认为0；
  • shadowBlur：模糊的像素数，默认0，即不模糊；

兼容性：
IE9、Firefox4、Opera11 最规范

15.2.8 渐变

渐变由 CanvasGradient 实例表示

  • createLinearGradient()：创建一个指定大小的线性渐变，并返回 CanvasGradient 对象的实例。
    参数（4）：起点的坐标 x 轴、y 轴、终点的坐标 x 轴、y 轴
  • addColorStop()：指定色标。
    参数（2）：色标位置（0（开始的颜色）到1（结束的颜色）之间的数字）、CSS 颜色值

或者使用函数确保坐标合适

  • createRadialGradient()：创建径向渐变（或放射渐变）。
    参数（6）：即起点圆的圆心、半径、终点圆的圆心、半径

15.2.9 模式

模式就是重复的图像，可以用来填充或描边图形。

createPattern()：创建一个新模式
参数（2）：一个HTML<img>元素、一个表示如何重复图像的字符串（”repeat”、”repeat-x”、”repeat-y”、”no-repeat”）。
第一个参数也可以是一个<vidio>元素或者另一个<canvas>元素。

15.2.10 使用图像数据

getImageData()：取得原始图像数据。返回的对象是 ImageData 的实例。
参数（4）：要取得其数据画面区域的 x 和 y 坐标，以及该区域的像素宽度和高度。

属性 说明
width 图像的宽度
height 图像的高度
data 是一个数组，保存每一个像素的数据。每个像素用4个元素保存：红、绿、蓝、透明度值

15.2.11 合成

应用到所有绘制操作的属性：

  • globalAlpha：介于0到1之间的值，用于所有绘制的透明度，默认值为0。
  • globalComposition-Operation：表示后绘制的图形怎样与先绘制的图形结合，值是字符串。
值 说明
source-over(默认) 后图位于先图上方
source-in 后图与先图重叠部分可见，两者其余部分透明
source-out 后图与先图不重叠部分可见，先图完全透明
source-atop 后图与先图重叠部分可见，先图不受影响
destination-over 后图位于先图下方，只有之前透明像素下的部分可见
destination-in 后图位于先图下方，两者不重叠的部分完全透明
destination-out 后图擦除与先图重叠部分
destination-atop 后图位于先图下方，在两者不重叠的地方，先图透明
lighter 后图与先图重叠部分值相加，使该部分变亮
copy 后图替换与之重叠的先图部分
xor 后图与先图重叠部分执行“异或”操作
