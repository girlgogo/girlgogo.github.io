20 章 JSON

JSON是一种数据格式，而不是一种编程语言，不支持变量、函数或对象实例。

20.1 语法

数据类型

  • 简单值：string、number、boolean、null（不支持undefined）
  • 对象：一组有序的键值对（包含简单值或复杂数据类型的值）
  • 数组：一组有序的值列表（可以包含任意类型，简单值、对象或数组）

20.1.1 简单值

注意：JS字符串和JSON字符串的区别
JSON字符串必需使用双引号 ！

20.1.2 对象

注意：手工编写JSON时的常见错误包括忘记给属性名添加双引号或写成了单引号。

JSON 对象和 JS 对象字面量的区别

  • 不用声明变量，因为没有变量的概念
  • 没有末尾的分号，因为不是JS语句
  • 对象的属性必需加双引号

20.1.3 数组

注意：JSON数组采用的就是JS的数组字面量形式，只不过没有变量和分号。

20.2 解析与序列化

JSON 流行的原因：

  • JSON 拥有与 JS 类似的语法
  • 可以把 JSON 数据结构解析为有用的 JS 对象。

20.2.1 JSON对象：ES5提供的解析JSON的工具

JSON 是 JS 语法的子集。因此 eval() 函数可以解析、解释并返回 JS 对象和数组。

  • ES5之前：eval()
  • ES5之后：JSON 对象

兼容性：IE8+ Firefox3.5+ Safari4+ Chrome Opera10.5+

JSON.stringify(jsData,[filter],[isIndent])：把JS对象转为JSON字符串
特点：

  • 默认情况下，输出的字符串不包含任何空格字符或缩进
  • 所有函数和原型成员以及值为 undefined 的属性都将被忽略，不体现在结果中

参数：可以转为JSON子串的JS数据类型，过滤器（数组或者函数），是否缩进

注意：序列化的内部顺序
（1）如果存在 toJSON() 方法而且能够通过它取得有效的值，则调用该方法。否则，按默认顺序执行序列化。
（2）如果提供第二个参数，应用这个函数过滤器。传入函数过滤器的值是第（1）步返回的值
（3）对第（2）步返回的每个值进行相应的序列化
（4）如果提供了第三个参数，执行相应格式化

JSON.parse(string,[replacer])：把 JSON 字符串解析为对应的 JS 类型（成为JS数据结构的子集）
特点：

  • 如果传递的字符串无效，该方法会抛出错误

参数：有效字符串, 还原函数

20.2.2 序列化选项

  1. 过滤结果
  • JSON.stringify(jsData,[filter],[isIndent])
    [filter]：过滤器
    如果是数组，结果中只包含数组中列出的属性；
    如果是函数，会对每一个键值对进行处理，函数的返回值就是对应键的值

  1. 字符串缩进
    [indent]：缩进，会插入一些换行符提高可读性
    如果是数字，代表缩进的空格数，数字>10会被自动设为10
    如果是非数值，会用作缩进字符，同样超过10个字符以外的字符不会被采用

  2. toJSON() 方法
    在需要序列化的对象内部自定义序列化方法，作为函数过滤器的补充

20.2.3 解析选项

  • JSON.parse(string, [replacer])
    [replacer]：还原函数，接收键和值两个参数，会对每个属性进行过滤
    如果还原函数返回undefined，则表示要从结果中删除相应的键
    如果返回其它的结果，则将该值插入到结果中
