6.1 React 概述

6.1.1 什么是 React

1、React 的开发背景
Facebook 需要解决的问题：构建数据不断变化的大型应用。

2、React 的特点

  • 简单：学习简单、代码简单
  • 声明式：自动 DOM 操作

React 的核心是组件，组件的设计目的是提高代码复用率、降低测试难度和代码复杂度.
1) 提高代码复用率：组件将数据和逻辑封装，类似面向对象中的类
2) 降低测试难度：组件高内聚低耦合，很容易对单个组件进行测试。
3) 降低代码复杂度：直观的语法可以极大提高可读性。

3、React 的发展过程
2013年6月，Facebook 官方发布 React
2013年9月，React 热度开始上涨
2015年3月，React Native 发布

6.3 React 组件生命周期详解

6.3.1 什么是生命周期

1、组件本质上是状态机，输入确定，输出一定确定。
状态发生转换时会触发不同的钩子函数，从而让开发者有机会做出响应。
可以用事件的思路来理解状态
组件的所有状态构成了组件的整个生命周期

6.3.2 初始化阶段介绍

1、初始化阶段可以使用的函数：

  • getDefaultProps：只调用一次，实例之间共享引用

    只有在组件的第一个实例初始化的时候会调用，然后 React 会把这个函数的返回结果保存起来，从第二个实例开始他们的默认属性都是这个统一的结果。在 JS 中有两种类型的数据，分别是值类型数据（数字、字符串、布尔值）和引用类型数据（对象、数组、函数）。

  • getInitialState：初始化每个实例特有的状态

    每个实例初始化的时候都会调用他；每个实例都有不同的状态，所以都会调用这个函数

  • componentWillMount：render 之前最后一次修改状态的机会

  • render：只能访问this.props 和this.state，只有一个顶层组件，不允许修改状态和 DOM 输出
  • componentDidMount：成功 render 并渲染完成真实 DOM 之后触发，可以修改 DOM

6.3.3 运行中阶段介绍

1、运行中阶段可以使用的函数：

  • componentWillReceiveProps：父组件修改属性触发（组件发生改变前触发），可以修改新属性、修改状态
  • shouldComponentUpdate：返回false会阻止 render 调用
  • componentWillUpdate：不能修改属性和状态
  • render：只能访问this.props 和this.state，只有一个顶层组件，不允许修改状态和 DOM 输出
  • componentDidUpdate：可以修改 DOM

6.3.4 销毁阶段介绍

销毁阶段可以使用的函数：

  • componentWillUnmount：在删除组件之前进行清理操作，比如计时器和事件监听器
