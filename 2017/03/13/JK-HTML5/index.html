1 HTML5

1.1 HTML 5与HTML 4的区别

1、推出的理由及目标

HTML5的意图是想要把目前Web上存在的各种问题一并解决掉，它的出现，对于Web来说意义重大。

  • Web浏览器之间的兼容性很低
  • 文档结构不够明确(比如div元素的应用)
  • Web应用程序的功能受到了限制

世界知名浏览器厂商对 html5 的支持：微软(IE9以上的版本支持)、Google、苹果、Opera、Mozilla

2、语法的改变

  • 内容类型（保持不变）
  • DOCTYPE声明（在html4中需声明是哪个版本，在html5中则不需要）
  • 指定字符编码
  • 可以省略标记的元素
  • 具有boolean值的属性
  • 省略引号

3、新增的元素和废除的元素以及新增的属性和废除的属性

新增的元素：

  • 新增的结构元素：section、article、aside 、header、hgroup、footer、nav、figure。
  • 新增的其他元素：video、audio、embed、mark、progress、meter、time、ruby、rt、rp、wbr、canvas、command、details、datalist、datagrid、keygen、output、source、menu
  • 新增的input元素的类型：email、url、number、range、Date Pickers

废除的元素：

  • 能使用CSS替代的元素：basefont、big、center、font、s、tt、u等
  • 不再使用frame框架
  • 只有部分浏览器支持的元素
  • 其他被废除的元素

新增的属性

  • 表单相关的属性
  • 链接相关的属性
  • 其他属性。

废除的属性

4、全局属性：可以对任何元素都使用的属性

  • contentEditable属性：微软开发，其他浏览器反编译并投入使用，规定是否可编辑元素的内容（true/false/classname（继承父元素的 contenteditable 属性））。

  • designMode属性：指定整个页面是否可编辑，只能在JavaScript脚本里被编辑（on/of）

  • hidden属性：通知浏览器不渲染该元素，使该元素处于不可见状态（true不可见/false可见）

  • spellcheck 属性：规定是否对元素内容进行拼写检查。可对以下文本进行拼写检查：类型为 text 的 input 元素中的值（非密码）、textarea 元素中的值和可编辑元素中的值。
    属性值：true 规定应当对元素的文本进行拼写检查 / false 不进行检查
    语法：<element spellcheck="value">

  • tabindex 属性：tabindex 属性规定元素的 tab 键控制次序（当 tab 键用于导航时）。
    语法：<element tabindex="number">
    注释：以下元素支持 tabindex 属性：<a>, <area>, <button>, <input>, <object>, <select> 以及 <textarea>。

1.2 HTML 5 新增的主体结构元素

1、article元素：

代表文档，页面或应用程序中独立的、完整的、可以独自被外部引用的内容。可以嵌套使用，可以用来表示插件。

2、section元素：

section 元素用于对网站或者应用程序页面中的内容进行分块。一个 section 元素常由内容及其标题组成。但是 section 元素并非一个普通的容器元素；当一个容器需要被直接定义样式或者通过脚本定义行为时，推荐使用 div 而非 section 元素。

注意：通常不推荐没有标题内容使用 section 元素；不要与 article 元素混淆

  • 不要将 section 元素作为设置样式的页面容器（div元素）。

  • 如果 article 元素、aside 元素、nav 元素更符合使用条件，那不要使用 section 元素； article 元素更强调独立性，可当成特殊的 section 元素。

  • 没有标题内容不要使用 section 元素。

3、nav元素：

nav 元素是一个可以用作页面导航的链接组，其中的导航元素链接到其他页面或当前页面的其他部分。并不是所有的链接组都要被放进 nav 元素，只需将主要的、基本的链接组放入nav元素即可。

nav 元素应用场景：

  • 传统导航条
  • 侧边栏导航
  • 页面导航
  • 翻页操作

注意：在 html5 中不能使用 menu 元素代替 nav 元素

4、aside元素：

aside元素用来表示当前页面或文章的附属信息部分，它可以包含与当前页面或主要内容相关的引用、侧边栏、广告、导航条，以及其他类似的有区别于主要内容的部分。

5、time元素和pubdate属性（具体发布时间）：

1.3 HTML 5 新增的非主体结构元素

1、header元素：

header 元素是一种具有引导和导航作用的结构元素，通常用来放置整个页面或者页面内的一个内容区块的标题，但是也可以包含其他内容，例如数据表格、搜索表单或相关的 logo 图片。
注意：header 可以出现多次。

2、footer元素和hgroup元素：

footer 元素可以作为其上层父级内容区块或者是一个根区块的脚注。footer 通常包括其相关区块的脚注信息，如作者、相关阅读链接及版权信息等。

hgroup 元素是将标题及子标题进行分组的元素。hgroup 元素通常会将 h1~H6 元素进行分组，譬如一个内容区块的标题及子元素算一组。

3、address元素和网页编码规则：

address 元素用来在文档中呈现联系信息，包括文档作者或文档维护者的名字、他们的网站链接、电子邮箱、真实地址、电话号码等。address 应该不只用来呈现电子邮箱或真实地址，还用来展现跟文档相关联的联系人的所有信息。

大纲编排规则：

  • 显示编排内容区域块：明确使用section等元素创建文档结构，在每个区域块使用标题，类似于h1~h6、hgroup等
  • 隐示编排内容区域块
  • 标题分级
  • 不同区域块可以使用相同级别的标题

1.4 HTML 5 表单新增元素和属性

1、form属性和formaction属性：

form 属性：在 html4 中，表单内的从属元素必须书写在表单内部，而在 html5 中，可以把他们书写在页面上任何地方，然后为该元素指定一个 form 属性，属性值为该表单的 id ，这样就可以声明该元素从属于指定表单了。

formaction 属性：在 html4 中，一个表单内的所有元素只能通过表单的 action 属性被统一提交到另一个页面，而在 html5 中可以为所有的提交按钮，增加不同的 formaction 属性，使单击不同的按钮时可以将表单提交到不同的页面。

2、formmethod和formenctype属性

formmethod属性：在html4中，一个表单内只能有一个action属性用来对表单内所有统一指定提交页面，所以每个表单也main内只有一个method属性来统一指定提交方法。在html5中，可以使用formmethod属性来对每一个表单元素分别指定不同的提交方法。

formenctype属性：在html4中，表单元素具有一个enctype属性，该属性用于指定在表单发送到服务器之前应该如何对表单内的数据进行编码。在html5中，可以使用formenctype属性对表单元素分别指定不同的编码方式。有3种形式【text/plain(表单数据中的空格转化成+，但不对特殊字符进行编码）、multipart/form-data（不对字符进行编码，在使用包含文件上传空间的表单时必须使用该值）、application/x-www-form-urlencoded（发送前编码所有字符，当表单元素的action属性为get时，浏览器则用当前的编码方式把表单数据转
化成字符）】

3、formtarget属性和autofocus属性：

formtarget：在 html4 中表单元素具有一个 target 属性，该属性用于指定在何处打开表单提交后所需要加载的页面。在 html5 中可以对多个提交按钮分别使用 formtarget 属性指定提交后在何处打开所需加载的页面。【属性有5种：-blank（在新浏览器页面中打开）、-self（在相同的框架的frame打开）、-parent（在当前框架的父框架中打开）、-top（在当前窗口中打开）、framename（在指定框架中打开）】

autofocus：为文本框、选择框或者按钮控件加上 autofocus 属性，当页面打开时，该控件自动获得光标焦点。

4、required属性和labels属性：

required：html5 中新增的 required 属性可以应用在大多数输入元素上，在提交时，如果元素中内容为空白，则不允许提交，同时浏览器中显示信息提示文字。

labels：在html5中，为所有可使用标签的表单元素、button、select元素等，定义一个labels属性，属性值为一个Nodelist对象，代表该元 素所绑定的标签元素所构成的集合。

1.5 HTML 5 表单新增元素和属性（续）

1、controls属性与placeholder属性

标签的control属性：在 html5 中，可以在标签内部放置一个表单元素，并且通过该标签的 control 属性来访问该表单元素。

文本框的placeholder属性： placeholder 是指当文本框处于未输入状态时显示的输入提示。当文本框处于未输入状态且未获取光标焦点时，模糊显示输入提示文字。

2、list属性和AutoComplete属性

文本框的list属性：在html5中，为单行文本框增加了一个list属性，该属性的值为某个datalist元素的id。datalist元素也是html5中新增的元素，该元素类似于选择框，但是当用户想要设定的值不在选择列表之内时，允许自行输入。datalist元素本身并不显示，而是当文本框获得焦点时以提示输入的方式显示。


文本框的 autocomplete 属性：帮助输入所用的自动完成功能，是一个既节省输入时间又十分方便的功能。在 html5 之前，因为谁都可以看见输入的值，所以在安全方面存在缺陷，只要使用 AutoComplete 属性，安全性方面也可以得到很好的控制。【on,off, 】

3、pattern属性和SelectionDirection属性

文本框的pattern属性：在html5中，对input元素使用pattern属性，并且将属性值设为某个格式的正则表达式，在提交时会针对这些进行检查，检查其内容是否符合给定格式。当输入的内容不符合给定格式时，则不允许提交，同时在浏览器中显示信息提示文字，提示输入的内容必须符合给定格式。


文本框的SelectionDirection属性：这对input元素与textarea元素，html5增加了SelectionDirection属性。当用户在这两个元素中用鼠标选取部分文字时，可以使用该属性来获取选取方向。当用户正向选取文字时，该属性值为“forward”，当用户反向选取文字时，该属性值为“backward”。当用户没有选取任何文字时，该属性为“forward”。


4、indeterminate属性和image属性

复选框的indeterminate属性：对于复选框checkbox元素来说，过去只是选取与非选取这两种状态。在html5中，可以在JavaScript脚本代码中对该元素使用indeterminate属性，以说明复选框处于“尚未明确是否选取”状态。

1.7 HTML 5 增强的页面元素

增强的页面元素：figure、figcaption、details、summary、mark、progress、meter、ol、dl、cite、small

1.7.1 figure、figcaption、details、summary、mark

  • figure元素和figcaption元素：figure元素是一种组合元素，带有可选标题，用来表示网页上独立的内容，将其从网页上移除后，不会对网页上的其他内容产生影响。图片、统计图或者是代码示例，以及音频插件、视频插件、统计表格等；figcaption元素表示figure元素的标题，从属于figure元素，必须书写与其内部，会显示出来。可以书写在figure元素的内部及其他元素的前面或者是后面。一个figure元素最多只允许放置一个figcaption元素，允许放置多个其它元素。

  • detail元素和summary元素：detail元素用于标识该元素内部的子元素可以被展开收缩显示的元素。【true：打开状态，false：收缩状态。默认状态是false】。元素内部可以放置文字、表单、插件或者是统计图的详细数据；summary元素从属于detail元素,用鼠标单击summary元素中的文字内容时，detail元素中的所有子元素都会被展开或者是收从属。如果detail元素中没有summary元素，浏览器会提供默认文字以供单击。


  • mark元素：表示页面中需要突出显示或高亮显示的对于当前用户具有参考作用的一段文字。

1.7.2 progress元素和meter元素

  • progress元素：代表一个任务的完成进度。这个进度可以是不确定的，表示进度正在执行，或者不清楚当前工作量有没有完成，也可以用0-100当中的一个数字来代表进度的完成情况。

  • meter元素：表示规定范围内的数值量。

1.7.3 ol、dl、cite和small元素

  • ol元素：新增start、reversed属性。


  • dl元素：从新定义后，表示多个名字的列表项，每一项都包含一条或多条带有名字的dt元素，用来表示术语，dt元素后面紧跟一个或多个dd元素，在一个元素内不允许有相同名字的dt元素出现，不允许有重复的术语。

  • cite元素：表示作品的标题（一首歌、一篇文章等）。

  • small元素：标识小字印刷体的元素。通常使用到免费声明、版权、法律规范等相关法律文字等内容中，不允许用在页面主体内容中，只允许作额外信息内嵌在页面中使用。

1.8 HTML 5 编辑API之Range对象（一）

1.8.1 Range 对象基本概念

一个Range对象代表页面上的一段连续区域，通过Range对象，可以获取或修改网页上的任何区域。


1.8.2 Range 方法之 SelectNode 等方法

SelectNode、SelectNodeContents、deleteContents方法

  • SelectNode：用于range对象的起点，指定为某个节点的起点,将range对象的终点指定为该节点的终点，使range不代表的区域中包含该节点。该方法并不将该节点设置为高亮显示状态，如果想要选取一个设置的节点，需要使用selection对象的andRange方法。

  • SelectNodeContents：用于range对象的起点，指定为某个节点中所有内容的起点,将range对象的终点指定为该节点中所有内容的终点,使range代表的区域中包含该节点中的所有内容。

  • deleteContents：用于将range对象所包含的内容从页面中进行删除。

1.8.3 Range 方法之 setStartsetEnd 等方法

setStart、setEnd、setStartBefore、setStartAfter、setEndBefore、setEndAfter方法

  • setStart、setEnd：setStart用于将某个节点中的某处位置指定为Range对象所代表区域的起点位置；setEnd用于将某个节点中的某处位置指定为Range对象所代表区域的结束位置。

  • setStartBefore、setStartAfter、setEndBefore、setEndAfter：setStartBefore用于将某个节点的起点指定为Range对象所代表区域的起点位置；setStartAfter用于将某个节点的终点位置指定为Range对象所代表区域的起点位置；setEndBefore用于将某个节点的起点指定为Range对象所代表区域的终点位置；setEndAfter用于将某个节点的终点指定为Range对象所代表区域的终点位置。

1.9 HTML 5 编辑API之Range对象（二）

1.9.1 Range对象方法

cloneRange、cloneContents、extractContents方法
cloneRange方法：用于当前的 range 对象进行复制，该方法返回当前 range 的对象。

cloneContents：用于在页面中追加一段 html 代码，并且将 range 对象所代表区域中的 html 代码克隆到追加到 html 代码之后。

extractContents：用于 range 对象所代表区域中的 html 代码克隆到 docfragment对象中。

1.9.2 Range对象方法之insertNode、compareBoundaryPoints方法

insertNode方法：用于指定节点插入到某个range对象所代表的区域中，插入位置为range对象所代表区域的起点位置，如果该点已经存在了页面当中，那该节点被移动到range对象所代表区域的起点处。


compareBoundaryPoints方法：用于比较两个range对象的起点位置或终点位置。

1.9.3 Range对象方法之collapse、detach方法

collapse方法：用于range对象代表区域的终点移动到该区域的起点处，或将range对象代表区域的起点移动到该区域的终点处。

detach方法：释放range对象。

1.21 HTML 5 应用缓存与Web Workers

1.21.1 应用缓存

1、什么是应用程序缓存
HTML 5 引入应用程序缓存，这意味着 web 应用可进行缓存，并可在没有英特网连接时进行访问。

2、应用缓存的优势：
1)：离线浏览 – 用户可在应用离线时使用它们
2)：速度 – 已缓存资源加载得更快
3)：减少服务器负载 – 浏览器将只从服务器下载更新过或更改过的资源

3、实现缓存：
如需启用应用程序缓存，请在文档的<html>标签中包含 manifest属性，manifest 文件的建议文件扩展名是：“ .appcache”

4、Manifest 文件：
1)：CACHE MANIFEST - 在此标题下列出的文件将在首次下载后进行缓存
2)：NETWORK - 在此标题下列出的文件需要与服务器的来连接，且不会被缓存
3)：FALLBACK - 在此标题下列出的文件规定当页面无法访问时回退页面（比如404页面）

1.22 服务器推送事件

1.22.1 服务器推送事件 - 介绍

1、HTML 5 服务器推送事件介绍：

服务器推送事件（Server-sent Events）是 HTML 5 规范中的一个组成部分，可以用来从服务端实时推送数据到浏览器

2、传统的服务器推送数据技术：

WebSocket： WebSocket 规范是 HTML 5 中的一个重要组成部分，已经被很多主流浏览器所支持。WebSocket 使用的是套接字连接，基于TCP协议。使用 WebSocket 之后，实际上是在服务器和浏览器之间建立一个套接字连接，可以进行双向的数据传输。功能强大，使用灵活，可以适用于不同的场景，但是技术比较复杂。

HTTP协议：简易轮询，即浏览器端定时向服务器端发出请求，来查询是否有数据更新。做法简单，不过轮询的时间需要进行仔细考虑。
